# Web Marker 产品路线图与改造计划

## 项目定位

**一句话定位**：把任意网页（本地或在线）变成可高亮、可标注、可演示的交互画布。

**核心差异化**：不做"又一个网页高亮工具"，而是瞄准三个被现有产品忽视的场景——
1. AI 生成的本地 HTML 文件的阅读标注
2. 任意网页上的实时演示标注（类似 PPT 激光笔/画笔）
3. 标注结果的分享与协作

### Phase 1：Chrome 扩展 + 在线网页支持

> 目标：从油猴脚本升级为独立 Chrome 扩展，覆盖在线网页场景，打好技术基座。

#### 1.1 扩展基础架构

```
WebMarker/
├── manifest.json          # Manifest V3
├── background/
│   └── service-worker.js  # 后台服务（存储调度、消息中继）
├── content/
│   ├── marker.js          # 核心标注逻辑（从油猴脚本迁移）
│   └── marker.css         # 页面注入样式
├── sidebar/
│   ├── index.html         # 侧边面板 UI
│   ├── sidebar.js
│   └── sidebar.css
├── popup/
│   ├── index.html         # 扩展弹窗（快速开关/统计）
│   └── popup.js
├── icons/                 # 扩展图标
└── _locales/              # 国际化
```

- 迁移到 Manifest V3（Chrome 的新扩展标准）
- 使用 `chrome.sidePanel` API 替代自绘侧边栏，更原生
- 使用 `chrome.storage.local` 替代 `GM_setValue`，支持更大存储量
- Content Script 注入到所有页面（`<all_urls>` + `file://`）

#### 1.2 在线网页支持

- 匹配规则扩展为 `*://*/*` + `file:///*`
- 针对动态页面（SPA）增强锚定策略：
  - 主锚定：XPath + 字符偏移（沿用）
  - 辅助锚定：文本指纹（选中文本 + 前后 50 字符 hash）
  - 兜底策略：全文搜索匹配
- 处理页面动态加载：监听 `MutationObserver`，在 DOM 稳定后恢复标注
- URL 归一化：忽略 hash/query 变化，同一页面共享标注

#### 1.3 UI/UX 增强

- 悬浮按钮支持拖拽定位 + 位置记忆
- 侧边栏宽度可调（窄模式 48px / 宽模式 200px 含标注列表）
- 快捷键支持：`Alt+H` 高亮、`Alt+E` 擦除、`Alt+Shift+H` 开关面板
- 右键菜单集成（作为侧边栏的补充入口）
- 标注数量角标显示在扩展图标上

#### 1.4 标注管理面板

- 侧边栏宽模式下展示当前页面所有标注列表
- 点击列表项滚动到对应标注位置
- 支持按颜色/类型/时间筛选
- 支持编辑标注颜色、删除单条

**预计周期**：4-6 周

### Phase 2：演示模式（核心差异化能力）

> 目标：把网页变成演示画布。这是竞品几乎没有做好的场景，是最大的差异化点。

#### 2.1 演示工具集

| 工具 | 说明 |
|------|------|
| **画笔** | 自由手绘，可调粗细和颜色，使用 Canvas overlay 实现 |
| **直线 / 箭头** | 指示关系或方向 |
| **矩形 / 椭圆框选** | 圈出重点区域 |
| **激光笔** | 红色光点跟随鼠标，2 秒自动消失（演示神器） |
| **聚焦灯** | 页面变暗，仅鼠标周围区域高亮（Spotlight 效果） |
| **文字标签** | 在任意位置放置文字便签 |

#### 2.2 技术方案

```
┌─────────────────────────────────┐
│         Canvas Overlay          │  ← 画笔/箭头/框选渲染层
│    (position:fixed, 全屏覆盖)    │     pointer-events 按需切换
├─────────────────────────────────┤
│         SVG Overlay             │  ← 文字标签/可交互标注
├─────────────────────────────────┤
│       原始网页内容               │  ← 文本高亮/变色在此层
└─────────────────────────────────┘
```

- Canvas 层负责自由绘制（画笔、箭头、形状），性能好
- SVG 层负责可交互的标注元素（文字标签、便签），支持点击编辑
- 演示标注和文本标注分开存储，互不干扰
- 演示模式下 `pointer-events` 切到 Canvas 层，退出后恢复到页面

#### 2.3 录制与回放（进阶）

- 记录用户的标注操作序列（时间戳 + 动作）
- 支持"回放"标注过程，配合演示场景
- 可导出为 GIF / 视频片段

**预计周期**：6-8 周

### Phase 3：批注与协作

> 目标：从个人工具进化为协作工具，打开付费空间。

#### 3.1 批注系统

- 在任意标注上添加文字批注（Markdown 支持）
- 批注气泡 UI，hover 显示，click 展开编辑
- 支持 @提及（为后续协作铺路）
- 批注支持插入图片（粘贴截图）

#### 3.2 分享功能

- **快照分享**：将当前页面 + 标注导出为 PNG / PDF
- **链接分享**：生成一个带标注数据的短链接，接收者打开原页面时自动加载标注
  - 技术方案：标注数据序列化 → 压缩 → Base64 编码到 URL fragment
  - 或使用轻量后端存储标注 JSON（配合短链服务）
- **协作标注**：多人实时在同一页面上标注
  - 使用 WebSocket 或 WebRTC 同步标注操作
  - 每人分配不同颜色区分

#### 3.3 云同步

- 用户账号系统（支持 Google/GitHub OAuth 登录）
- 标注数据云端存储（按页面 URL 索引）
- 跨设备同步（多台电脑/浏览器共享标注）
- 标注历史版本管理

**预计周期**：8-12 周

### Phase 4：AI 增强（远期愿景）

> 目标：用 AI 能力做差异化，让工具更智能。

| 功能 | 说明 |
|------|------|
| **智能高亮** | 一键自动高亮页面关键信息/结论/数据 |
| **摘要生成** | 基于用户标注内容自动生成结构化摘要 |
| **标注建议** | 阅读时智能推荐可能需要标注的段落 |
| **多语言翻译** | 标注区域一键翻译 |
| **问答** | 基于页面内容 + 用户标注进行 Q&A |

技术选型：调用 OpenAI / Claude API，或集成浏览器端的本地小模型（如 Chrome 内置 AI API）。

## 竞争策略

### 不和巨头正面硬刚

Liner、Hypothesis 等已有大量用户的通用标注工具，不去抢他们的核心盘。

### 切入被忽视的缝隙市场

1. **AI 内容阅读者**：用 ChatGPT / Claude 生成 HTML 报告的人群，需要对本地文件做标注——现有工具几乎不支持
2. **在线授课/演示**：需要在任意网页上实时标注演示——没有轻量好用的解决方案
3. **内容创作者**：需要在参考网页上做标注、截图分享给读者/粉丝

### 增长飞轮

```
免费油猴脚本/扩展 → 口碑传播 → 用户量
        ↓                              ↓
  GitHub 开源社区              付费转化（演示/协作）
        ↓                              ↓
  开发者贡献/插件生态    →     收入 → 持续投入
```
